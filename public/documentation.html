Movie App!!

The objective of this project is to build a server-side component of a "movies" web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

BUSINESS LOGIC URL REQUEST PARAMETER HTTP METHOD REQUEST BODY DATA FORMAT RESPONSE BODY DATA FORMAT
1. Return a list of all movies to the user "/movies" none GET None A JSON object holding data about all the movies.
Example: Title: "A Clockwork Orange" Description: "In the future, a sadistic gang...etc." Genre: { Name: "Crime" Description: "Crime is a genre known for...etc." }, Director: { Name: "Stanley Kubrick" Birth: "1928" Death: "1999" }

2. Return data about a single movie by title to the user "/movies/:title" :title GET None A JSON object holding data about a single movie.
Example: Title: "A Clockwork Orange" Description: "In the future, a sadistic gang...etc." Genre: { Name: "Crime" Description: "Crime is a genre known for...etc." }, Director: { Name: "Stanley Kubrick" Birth: "1928" Death: "1999" }

3. Return data about a genre by name/title "/movies/genre/:genreName" :name GET None A JSON object holding data about a genre by name.
Example: Genre: { Name: "Crime" Description: "Crime is a genre known for...etc." }

4. Return data about a director by name "/movies/:directors/:directorName" :name GET None A JSON object holding data about a director by name.
Example: Director: { Name: "Stanley Kubrick" Birth: "1928" Death: "1999" }
5. Allow new user to register "/users" none POST
A JSON object holding data about the new user infos to add.
Example:
Name: "Raspberry" Email: "raspberry@gmail.com" Password: "iamnotastrawberry" Birthday: "1994-07-23" FavoriteMovies: { "A Clockwork Orange" "Dancer in the Dark" "Wild at Heart" }

A JSON object holding data about the new infos added.
Example: { _id: "62228b13dec0e2bdfd000ff3" Name: "Raspberry" Email: "raspberry@gmail.com" Password: "iamnotastrawberry" Birthday: "1994-07-23" FavoriteMovies: { "A Clockwork Orange" "Dancer in the Dark" "Wild at Heart" }
6. Allow users to update their infos "/users/:id" :username PUT A JSON object holding data about the new username to add.
Example: { Name: "Potato" Email: "potato@gmail.com" Password: "iamnotabeetroot" Birthday: "2001-06-21" FavoriteMovies: "Melancholia" } 		A JSON object holding updated data about the user.
Example: { _id: "62228b13dec0e2bdfd000ff4", Name: "Potato" Email: "potato@gmail.com" Password: "iamnotabeetroot" Birthday: "2001-06-21" FavoriteMovies: "Melancholia" }
7. Allow users to add a movie to their list of favorites (showing only a text that a movie has been added) "/users/:id/movies/:movieTitle" :id, :movieTitle PATCH None A message confirming the movie has been added to the list of favorite movies.
Example: "Fantastic Planet has been added to the list of favorite movies"
8. Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed) "/users/:id/movies/:movieTitle" :id, :movieTitle DELETE None A message confirming the movie has been removed from the list of favorite movies.
Example: "Claire's Knee has been deleted from the list of favorite movies"
9. Allow existing users to deregister (showing only a text that a user email has been removed) "/users/:id" :username DELETE None A message confirming the user has been removed from the list of users.
Example: "Raspberry has been removed from the list of users"
