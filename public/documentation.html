Movie App!!

The objective of this project is to build a server-side component of a "movies" web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

BUSINESS LOGIC URL REQUEST PARAMETER HTTP METHOD REQUEST BODY DATA FORMAT RESPONSE BODY DATA FORMAT
1. Return a list of all movies to the user "/movies" none GET None A JSON object holding data about all the movies.
Example: { Title: "Ice Age", Year: "2002", Genre: { Name: "Comedy", Description: "Comedy films are films that make the audience laugh without making them think about existential questions" }, Director: { Name: "Chris Wedge", Birth: "1957", Death: "-" }
2. Return data about a single movie by title to the user "/movies/:title" :title GET None A JSON object holding data about a single movie.
Example: Title: "Ice Age"
3. Return data about a genre by name/title "/movies/genre/:name" :name GET None A JSON object holding data about a genre by name.
Example: Genre: { Name: "Comedy", Description: "Comedy films are films that make the audience laugh without making them think about existential questions" },
4. Return data about a director by name "/movies/:directors/:name" :name GET None A JSON object holding data about a director by name.
Example: Director: { Name: "Chris Wedge", Birth: "1957", Death: "-" }
5. Allow new user to register "/users" none POST A JSON object holding data about the new user infos to add.
Example: { "name": "Jerome", "favoriteMovies": [], "email": "mashedpotato@gmail.com", "password": "ialsolovefries", "birthday": "01/01/2001", } 		A JSON object holding data about the new infos added.
Example: { "name": "Jerome", "favoriteMovies": [], "email": "mashedpotato@gmail.com", "password": "ialsolovefries", "birthday": "01/01/2001", "id": "7a8e11f7-ce31-4928-8272-0187fb9f8179" }
6. Allow users to update their user name "/users/:username" :username PUT A JSON object holding data about the new username to add.
Example: { "name": "UNmashedpotatoes" } 		A JSON object holding data about the new user to add with the new name.
Example: { "id": 1, "username": "mashedpotato", "email": "mashedpotato@gmail.com", "password": "ialsolovefries", "birthday": "01/01/2001", "favoriteMovies": [ "Ice Age", "Finding Nemo", "Lion King" ], "name": "UNmashedpotatoes"
7. Allow users to add a movie to their list of favorites (showing only a text that a movie has been added) "/users/:username/movies/:movieID" :username, :movieID POST A JSON object holding data about the movie added to the list of favorite movies. A text showing that the movie has been added to their list of favorite movies.
8. Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed) "/users/:username/movies/:movieID" :username, :movieID DELETE None A text showing that the movie has been removed to their list of favorite movies.
9. Allow existing users to deregister (showing only a text that a user email has been removed) "/users/:username" :username DELETE None A text showing that the user email has been removed.
