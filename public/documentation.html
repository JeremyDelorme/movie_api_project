My Movie API

The objective of this project is to build a server-side component of a "movies" web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

CLIENT REQUEST URL HTTP METHOD REQUEST BODY DATA FORMAT RESPONSE BODY DATA FORMAT
1.Return a list of all movies to the user /movies GET None A JSON object holding data about all the movies.
Example: Title: "A Clockwork Orange" Description: "In the future, a sadistic gang...etc." Genre: { Name: "Crime" Description: "Crime is a genre known for...etc." }, Director: { Name: "Stanley Kubrick" Birth: "1928" Death: "1999" }


2.Return all data (description, genre, director, image URL, whether it's featured or not) about a single movie by title to the user /movies/:title GET None A JSON object holding data about a single movie.
Example: Title: "A Clockwork Orange" Description: "In the future, a sadistic gang...etc." Genre: { Name: "Crime" Description: "Crime is a genre known for...etc." }, Director: { Name: "Stanley Kubrick" Birth: "1928" Death: "1999" }

3.Return data about a genre (description) by Name /movies/genre/:Name GET None A JSON object holding data about a genre by name.
Example: Genre: { Name: "Crime" Description: "Crime is a genre known for...etc." }

4.Return data about a director (bio, birth, year, death year) by Name /movies/director/:Name" GET None A JSON object holding data about a director by name.
Example: Director: { Name: "Stanley Kubrick" Birth: "1928" Death: "1999" }
5.Allow new users to register (Username, Password and Email are required fields) /users POST
A JSON object holding data about the new user infos to add.
Example:
Username: "Potato" Email: "potato@gmail.com" Password: "iamnotabeetroot" Birthday: "2001-06-21" FavoriteMovies: { "A Clockwork Orange" "Dancer in the Dark" "Wild at Heart" }

A JSON object holding data about the new infos added.
Example: { _id: "62228b13dec0e2bdfd000ff3" Name: "Raspberry" Email: "raspberry@gmail.com" Password: "iamnotastrawberry" Birthday: "1994-07-23" FavoriteMovies: { "A Clockwork Orange" "Dancer in the Dark" "Wild at Heart" }
6.Allow users to update their user infos (user is found by username) /users/:Username" PUT A JSON object holding data about the new username to add.
Example: { Password: "iamnotastrawberry" } 		A JSON object holding updated data about the user.
Example: { Password: "iamnotablueberry" }
7. Allow users to add a movie to their list of favorite movies "/users/:Username/movies/:MovieID" PATCH None A message confirming the movie has been added to the list of favorite movies.
Example: "Fantastic Planet has been added to the list of favorite movies"
8. Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed) "/users/:Username/movies/:MovieID" DELETE None A message confirming the movie has been removed from the list of favorite movies.
Example: "Claire's Knee has been deleted from the list of favorite movies"
9. Allow existing users to deregister (showing only a text that a user email has been removed) "/users/:Username" DELETE None A message confirming the user has been removed from the list of users.
Example: "Raspberry has been removed from the list of users"
